## 📋 項目概述

這是一個為FFXIV玩家設計的**全功能狩獵怪物管理工具**，具備：
- ✅ 快速搜尋 & 多條件篩選
- ✅ 實時雲端同步（Firebase Firestore）
- ✅ 自訂分組追蹤看板
- ✅ 批量管理怪物
- ✅ 職業分類系統
- ✅ 多裝置無縫訪問

## ✨ 功能特色

### 🎯 狩獵搜尋板

| 功能 | 說明 |
|------|------|
| **全局搜尋** | 按名稱即時查找怪物 |
| **版本篩選** | 支援 2.0～7.0 所有擴展版本 |
| **地圖篩選** | 依版本自動載入對應地圖清單 |
| **等級與FATE** | 快速定位特定等級或FATE怪物 |
| **職業篩選** ⭐ | 按討伐筆記職業標籤篩選（12職業 × 50編號） |
| **座標複製** | 一鍵複製：怪物名稱 + 地圖 + 座標 |
| **展開/摺疊** | 批量切換座標可見性 |

### 📌 追蹤看板

| 功能 | 說明 |
|------|------|
| **自訂分組** | 建立多個分類分組（預設「常用區域」） |
| **拖曳管理** | 滑鼠拖曳怪物卡片在分組間移動 |
| **獨立篩選** | 各板獨立的版本/地圖/職業篩選 |
| **批量展開** | 全部展開/摺疊追蹤怪物座標 |
| **快速移除** | Hover時顯示刪除按鈕 |

### 🛠️ 管理控制台

| 功能 | 說明 |
|------|------|
| **新增單筆** | 設定名稱、版本、等級、FATE、職業、座標 |
| **批量新增** ⭐ | 一次新增多隻怪物，支援預設配置 |
| **批次座標解析** | 自動識別 `X:數字 Y:數字` 格式 |
| **表格管理** | 檢視、編輯、刪除所有怪物 |
| **批量選擇** | 高效率編輯管理 |

## 🎮 功能詳解

### 1. 狩獵搜尋板使用指南

```
步驟 1: 在搜尋欄輸入怪物名稱
       ↓
步驟 2: 使用篩選器縮小範圍
       • 版本 → 地圖（自動更新）
       • 等級 / FATE 狀態
       • 職業（可選）
       ↓
步驟 3: 點擊卡片展開座標
       ↓
步驟 4: 複製座標或Pin到追蹤看板
```

**💡 職業篩選小技巧**：
- 點擊「討伐筆記」按鈕 → 顯示藍色 & 下拉選單
- 預設「全部」→ 顯示**所有有職業標籤**的怪物
- 選擇特定職業 → 只顯示**該職業**的怪物
- 再次點擊 → 取消篩選

### 2. 追蹤看板操作

```
✓ 使用獨立篩選器搜尋與過濾
✓ 拖曳怪物卡片在分組間移動
✓ Hover時點擊「×」移除追蹤
✓ 點擊分組名稱編輯名稱
✓ 「+」按鈕新增自訂分組
```

### 3. 管理控制台操作

#### 📝 新增單筆怪物

```
1. 點擊 [新增怪物] → 打開編輯對話框
2. 填入基本資訊：
   • 怪物名稱 ✓
   • 版本 ✓
   • 等級 (B/A/S/SS/None)
   • FATE 狀態 (是/否)
3. 新增座標（可多筆）：
   • 使用「批次座標解析」快速匯入
     格式：X:12.5 Y:34.6 (自動識別)
   • 或手動新增
4. 設定職業標籤（可選）
   • 點擊「討伐筆記」激活職業選擇
   • 預設「劍術士01」
5. 點擊 [儲存] ✓
```

#### 🔥 批量新增怪物（新功能）

```
1. 點擊 [批量新增列表] → 打開批量窗口
   
2. 設定共用配置
   ✓ 版本（自動讀取篩選值）
   ✓ 地圖（自動讀取篩選值）
   ✓ 等級
   ✓ FATE 狀態
   ✓ 討伐筆記職業（預設劍術士01）

3. 在「名稱輸入」區域貼入名稱清單
   格式：一行一個怪物名稱
   
4. 點擊 [解析名稱 →] 預覽結果
   （實時顯示所有標籤配置）

5. 確認無誤 → 點擊 [確認批量新增]
   系統自動寫入 Firestore ✓
```

**職業操作速查**：

| 操作 | 按鈕狀態 | 下拉選單 | 結果 |
|------|---------|--------|------|
| 初始 | 灰色 | 隱藏 | (無職業) |
| 點擊激活 | 藍色 | 顯示 | 設置職業0 |
| 選擇職業 | 藍色 | 顯示具體職業 | 綁定該職業 |
| 再次點擊 | 灰色 | 隱藏 | 清除職業 |

#### ✏️ 編輯現有怪物

```
1. 在表格找到目標怪物
2. 點擊 [Edit] → 打開編輯對話框
3. 修改任何字段：
   • 名稱、版本、等級、FATE
   • 座標（新增/刪除）
   • 職業標籤（同新增單筆邏輯）
4. 點擊 [儲存] ✓
5. 關閉時職業狀態自動重置
```

#### 🗑️ 刪除怪物

```
1. 在表格找到目標怪物
2. 點擊 [Trash] 按鈕
3. 確認刪除提示 → 完成 ✓
```

## 📦 數據結構

### 怪物 (Monster)

```javascript
{
  id: string,             // 唯一識別碼 "m_timestamp_random"
  name: string,           // 怪物名稱
  version: string,        // 版本 "2.0" ~ "7.0"
  rank: string,           // 等級 "None" | "B" | "A" | "S" | "SS"
  isFate: boolean,        // 是否為FATE怪物
  job: string | null,     // 職業標籤 "劍術士01" or null
  locations: Array[
    {
      map: string,        // 地圖名稱
      x: string,          // X座標
      y: string           // Y座標
    }
  ]
}
```

### 用戶PIN (User Pin)

```javascript
{
  id: string,                // 怪物ID
  groupId: string,          // 所屬分組ID
  updatedAt: number         // 最後更新時間戳
}
```

### 自訂分組 (Group)

```javascript
{
  id: string,               // 分組ID
  name: string,            // 分組名稱
  order: number,           // 排序順序
  canDelete: boolean       // 是否可刪除
}
```

### Firebase 路徑結構

```
Firestore
└── artifacts/{appId}/
    ├── public/
    │   └── data/monsters/          ← 全局怪物清單
    └── users/{userId}/
        ├── pins/                    ← 用戶PIN追蹤
        └── groups/                  ← 用戶自訂分組
```

## ❓ 常見問題

### 💬 Q: 追蹤板項目不會按帳號分類儲存？

**A:** 這個問題已修正，請確認你在最新版中已執行以下步驟：
- 重新登入帳號
- 確認右上角顯示正確 `userStore.virtualId`
- 針對每個帳號在 `artifacts/{appId}/users/{userId}/pins` 下可見一致資料

若你仍遇到：
- [ ] 追蹤項目漏寫或存錯帳號，請先清除 localStorage 再重試
- [ ] 若跨帳號切換持續錯亂，請回報開發者以便一併檢查 `useUserPinsStore` 初始化邏輯


### 💬 Q: 怪物未顯示？

**A:** 檢查以下項目：
- [ ] 版本篩選是否正確
- [ ] 地圖篩選是否正確
- [ ] Firebase 連線狀態
- [ ] 清空搜尋欄重試
- [ ] 重新登入帳號

### 💬 Q: 座標複製失敗？

**A:** 檢查以下項目：
- [ ] 瀏覽器剪貼簿權限
- [ ] 座標欄位已完整填寫
- [ ] 嘗試刷新頁面
- [ ] 在另一個瀏覽器嘗試

### 💬 Q: 批量新增無反應？

**A:** 嘗試以下步驟：
- [ ] 確認已解析名稱清單（預覽顯示）
- [ ] 檢查網路連線狀態
- [ ] 等待 Firestore 完成寫入（可能需要 2~5 秒）
- [ ] 檢查瀏覽器開發工具的 Console 標籤是否有錯誤

### 💬 Q: 職業標籤無法保存？

**A:** 確認以下操作順序：
1. 點擊「討伐筆記」按鈕（應變藍色）
2. 選擇職業與編號
3. 點擊「儲存」前確認下拉選單顯示正確職業
4. 提交前檢查預覽區域

### 💬 Q: 分組編輯不成功？

**A:** 嘗試以下方法：
- [ ] 確認分組 `canDelete` 為 `true`（預設分組無法刪除）
- [ ] 刪除分組前移出該分組的所有怪物
- [ ] 右鍵/長按檢查是否有編輯菜單
